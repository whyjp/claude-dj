동작 방식

네 단계

01

Hook 이벤트 수신

Claude Code가 PermissionRequest hook을 발화. 일반 tool이면 BINARY, AskUserQuestion이면 CHOICE로 분기. Bridge Server가 수신.

02

버튼 레이아웃 전환

세션 state에 따라 D200 버튼 레이아웃 즉시 교체. BINARY면 ✅❌, CHOICE면 1~N 번호 버튼. PROCESSING이면 웨이브 애니메이션.

03

물리 버튼 입력

화면으로 내용 확인 후 버튼을 누름. 포커스 이동 없음. 타이핑 없음.

04

응답 전달

Bridge가 결정을 Hook stdout JSON으로 반환. 일반 permission이면 allow/deny, AskUserQuestion이면 updatedInput.answer. Claude Code 즉시 다음 단계.

세션 상태

State가 버튼을 결정한다

세션 state가 바뀌는 순간 버튼 레이아웃 전체가 교체된다. PROCESSING에서는 버튼들이 웨이브로 흘러 작업 중임을 시각화한다.

IDLE

대기

완료 또는 프롬프트 대기. 전체 dim. 눌림 무반응.

PROCESSING

작업 중

PreToolUse 감지. 버튼들이 offset 딜레이로 웨이브 점멸. "뭔가 하고 있다" 시각화.

WAITING_BINARY

Permission 대기

PermissionRequest 수신. ✅ Approve와 ❌ Deny만 활성. 나머지 dim.

Bridge ↔ Plugin 메시지

Bridge Server와 Ulanzi Plugin(또는 Virtual D200 FE)이 주고받는 메시지 스키마.

← Bridge → Plugin: 레이아웃 전체 설정
{ "type": "LAYOUT", "preset": "binary" | "choice" | "processing" | "idle", "session": { "id": "abc", "name": "api-server", "state": "…" }, // binary preset "prompt": { "toolName": "Bash", "command": "rm -rf dist", "hasAlwaysAllow": false }, // choice preset "choices": [ { "index": 1, "label": "Refactor module" }, { "index": 2, "label": "Fix failing tests" } ] }
← Bridge → Plugin: 개별 업데이트
{ "type": "ALL_DIM" } { "type": "BUTTON_STATE", "slot": 0, "state": "approve" | "deny" | "always" | "processing" | "empty", "text": "" } { "type": "BUTTON_CHOICE", "slot": 0, "choiceIndex": 0, "number": 1, "label": "Fix tests" } { "type": "SESSION_COUNT", "total": 3, "waiting": 1 } { "type": "SESSION_NAME", "name": "api-server" } // BIG_WINDOW 없음 — 시스템 전용, 제어 불가
→ Plugin → Bridge: 버튼 입력
{ "type": "BUTTON_PRESS", "slot": 0, "timestamp": 1743000000000 } { "type": "PLUGIN_READY", "version": "0.1.0" } // BIG_WINDOW_PRESS 없음 — 시스템 전용, 클릭 불가 // slot 11 클릭 → BUTTON_PRESS slot:11 (세션 전환, Phase 2)
BUTTON_PRESS → Bridge가 slot을 decision으로 매핑.
slot 11 클릭 → 세션 전환 (Phase 2)
← AskUserQuestion Hook stdin
// PermissionRequest hook에서 수신 // tool_name: "AskUserQuestion" 판별 { "hook_event_name": "PermissionRequest", "tool_name": "AskUserQuestion", "tool_input": { "question": "How to proceed?", "options": [ { "label": "1", "description": "Refactor" }, { "label": "2", "description": "Fix tests" }, { "label": "3", "description": "Show diff" } ] } } // 응답: updatedInput.answer = "2"
tool_name === "AskUserQuestion" 판별 → CHOICE 분기.
응답: updatedInput.answer = "2" (선택 번호)
